# Data Directory

This directory contains video data for the fish migration monitoring project.

## Directory Structure

```
data/
├── raw/           # Original, unprocessed video files
├── processed/     # Processed video clips or frames
└── annotations/   # Manual annotations for training
```

## Raw Data (`raw/`)

Place your original video files here. Supported formats:
- MP4
- AVI
- MOV
- Other formats supported by OpenCV

**Note**: Video files are large and should not be committed to the repository. The `.gitignore` file is configured to exclude video files from version control.

### Organizing Raw Data

Recommended naming convention:
- `location_date_time.mp4`
- Example: `riverA_20240315_0900.mp4`

Consider creating subdirectories for different:
- Locations
- Time periods
- Camera setups

## Processed Data (`processed/`)

This directory stores:
- Extracted frames
- Video clips
- Pre-processed videos

Generated by preprocessing scripts or manual extraction.

## Annotations (`annotations/`)

Store annotation files for training detection models:
- Bounding box coordinates
- Fish species labels
- Image/frame metadata

### Annotation Format

Common formats:
- **YOLO format**: Text files with normalized coordinates
- **COCO format**: JSON with image and annotation data
- **Pascal VOC**: XML files with bounding boxes

### Annotation Tools

Recommended tools for creating annotations:
- [LabelImg](https://github.com/tzutalin/labelImg) - For YOLO/Pascal VOC
- [CVAT](https://github.com/openvinotoolkit/cvat) - Web-based video annotation
- [Label Studio](https://labelstud.io/) - Flexible annotation platform

## Data Management Best Practices

### Version Control
- **DO NOT** commit large video files to Git
- Use Git LFS for model files if needed
- Document data sources and versions in a separate file

### Storage
- Keep backups of original videos
- Use external storage or cloud storage for large datasets
- Compress videos if storage is limited

### Privacy and Ethics
- Ensure you have permission to use video data
- Remove any sensitive or identifying information
- Follow institutional guidelines for data handling

## Getting Sample Data

If you need sample data for testing:
1. Record a short video (1-2 minutes)
2. Use publicly available fish migration videos
3. Check with your project advisor for existing datasets

## Data Statistics

Keep track of your dataset:
- Total number of videos
- Total duration
- Number of annotated frames
- Number of fish instances
- Date ranges covered

Consider creating a `data_inventory.csv` file to track this information.
